# Deep Web Research (3-서브에이전트 병렬 검색 + 취합)

복합 주제를 단일 검색이 아니라 **서로 다른 검색 각도 3개로 병렬
조사**하고 그 결과를 교차 검증·취합해 답한다. 검색 일은
`web-searcher` 서브에이전트에게 `task` 도구로 위임한다.

## 사용 시점

- 사용자가 "깊게/심층/여러 관점으로 조사" 를 **명시적으로** 요청할 때
- 주제가 다면적이라 단일 쿼리로는 한쪽만 보게 될 때

단순 사실 1건 확인은 이 스킬을 쓰지 말고 기본 `web_search` 로 처리한다.

## 절차

### 1단계 — 검색 각도 3개 분할

주제를 **서로 겹치지 않는 3개의 검색 각도**로 나눈다.

> TODO(learning): 분할 축 정책을 확정하라. 후보:
> - (a) 관점축: 찬성근거 / 반대근거 / 중립·통계
> - (b) 시간축: 배경·역사 / 최신 동향 / 향후 전망
> - (c) 구조축: 정의·개념 / 사례·데이터 / 한계·반론
>
> 어떤 축이 "취합 가치(서로 다른 정보가 모임)"를 극대화하는지는
> 도메인 판단이다. 아래는 (a) 골격 — 정책 확정 후 이 섹션을 교체.
> 현재 골격: 기본은 (a) 관점축, 주제가 시의성 강하면 (b) 시간축.

분할한 3개 각도를 각각 한 문장 검색 지시로 적는다.

### 2단계 — task 3회 위임 (병렬)

`task` 도구를 **한 턴에 3번 호출**한다(subagent_type="web-searcher").
각 호출의 지시(description/입력)에 1단계의 각도 1개씩을 넣는다.
한 턴에 3개 tool_call 을 함께 emit 하면 동시 실행된다.

각 task 입력에 반드시 포함:
- 조사할 검색 각도(1단계 결과 1개)
- "사실 1줄 + 출처 URL 1줄 형식으로, 추측 없이, 근거 없으면 명시"

### 3단계 — 교차 취합

3개 서브에이전트 결과를 받으면:

1. 3개에서 **공통으로 확인된 사실** → 신뢰도 높음으로 분류
2. **한 곳에서만 나온 사실** → "단일 출처" 표시 후 포함
3. **상충하는 주장** → 양쪽 모두 출처와 함께 병기, 단정 금지
4. 최종 답변은: 요약 → 핵심 사실(출처 인용) → 상충/불확실 지점 순

추측으로 빈틈을 메우지 않는다. 3개 결과로 답할 수 없으면 그렇게 말한다.
